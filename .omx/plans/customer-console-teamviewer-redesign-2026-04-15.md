# 고객 화면 재구성 계획 — TeamViewer 스타일 데이터 콘솔

작성일: 2026-04-15

## 요구 요약
- 고객 화면을 카드/인스펙터 느낌보다 **데이터 관리 콘솔**처럼 보이게 재구성한다.
- 메인 영역은 고객을 행 단위로 길게 나열하는 **밀도 높은 테이블**로 바꾼다.
- 상세는 우측에서 열리는 **토글형 상세 드로어/패널**로 바꾼다.
- 현재 구조를 억지로 유지하지 않고, 사용자가 말한 방향(목록 중심 + 우측 세부 열림)으로 재설계한다.
- 이번 산출물은 계획만 포함하며 코드는 작성하지 않는다.

## 목표 UX
1. 화면을 열자마자 “고객 데이터를 관리하는 화면”이라는 인상을 준다.
2. 한 화면에서 여러 고객 상태를 빠르게 훑을 수 있어야 한다.
3. 상세는 필요할 때만 우측에서 열어 맥락 전환 없이 확인/수정한다.
4. 상태/다음 조치가 각 행에서 바로 보여야 한다.
5. 생성/편집/이력 확인이 동일한 상세 드로어 패턴 안에서 이뤄져야 한다.

## 권장 구조

### 1) 상단: 얇은 운영 헤더
- 현재 큰 제목 + 칩 조합은 유지 비중을 낮춘다.
- 고객 탭 전용 상단은 아래처럼 단순화한다.
  - 좌측: `고객 관리` 제목 + 총 고객 수/마감 필요 수 요약
  - 우측: `새 고객`, `인증서 일괄 점검`, 필요 시 `필터 초기화`
- 글로벌 액션바(`web/src/App.tsx`)는 남기되, 고객 탭에서는 칩 수를 줄이거나 더 관리형 지표로 정리한다.

### 2) 본문 메인: 데이터 테이블 우선
- 현재 카드형 목록 대신 진짜 테이블형 스캔 UI로 전환한다.
- 추천 컬럼:
  1. 고객명/상호
  2. 사업자번호
  3. 팝빌 상태
  4. 인증서 상태
  5. 발행 방식
  6. 최근/다음 조치
  7. 종합 상태
- 상태는 텍스트 + 배지 혼합으로 짧게 표시한다.
- 행 클릭 시 우측 드로어가 열린다.
- 행 우측 마지막 컬럼은 `열기` 또는 `빠른 조치`만 남겨 과한 버튼 밀도를 줄인다.

### 3) 우측: 토글형 상세 드로어
- 기본 추천: **데스크톱에서는 도킹형 우측 드로어**, 좁은 화면에서는 **오버레이 드로어**.
- 동작:
  - 행 클릭 → 드로어 열림
  - 닫기 버튼/chevron → 드로어 닫힘
  - 새 고객 버튼 → 동일 드로어를 생성 모드로 열기
- 드로어 내부 구성:
  - 헤더: 고객명, 사업자번호, 핵심 상태 배지, 1차 액션
  - 섹션 A: 상태 요약(팝빌/인증서/발행 방식/주소)
  - 섹션 B: 해결 필요 항목(체크리스트)
  - 섹션 C: 기본 정보 편집
  - 섹션 D: 발행 이력
  - 섹션 E: 저빈도 작업/위험 작업
- TeamViewer 느낌처럼 오른쪽 패널은 “선택한 행의 컨텍스트 패널”로 보이게 한다.

## 정보 구조 재편 원칙
- 지금처럼 “왼쪽 목록 + 오른쪽 큰 카드형 상세”가 아니라,
  **왼쪽은 스캔용 표 / 오른쪽은 선택 객체 컨텍스트**로 역할을 분리한다.
- 목록에서 보여줄 정보와 상세에서만 보여줄 정보를 분명히 나눈다.
- 상세에서만 필요한 긴 설명/폼/저빈도 작업은 드로어 내부 accordion으로 정리한다.

## 상태 모델 계획
- 기존 상태 자산은 최대한 재사용한다.
  - `customerListFilter`, `customerSearchQuery`, `selectedCustomer`, `selectedCustomerReadiness`, `selectedCustomerIssues`, `selectedCustomerIssuedDrafts`
- 추가될 가능성이 큰 UI 상태:
  - `customerDrawerOpen: boolean`
  - `customerTableViewMode: "all" | "attention" | "ready"` 정도의 단순 뷰 프리셋
  - 필요 시 `customerSortKey`
- 핵심은 데이터 계산 로직은 `App.tsx`에 두고, 화면 조합만 고객 탭에서 바꾸는 것이다.

## 구현 단계

### 단계 1. 구조 분해
- `CustomersTab.tsx`의 현재 단일 대형 파일을 화면 역할 기준으로 분해한다.
- 권장 단위:
  - 고객 툴바
  - 고객 데이터 테이블
  - 고객 상세 드로어
  - 고객 폼 섹션
- 목적: 재구성 폭이 큰 만큼, 구조를 먼저 나눠야 이후 수정이 쉬워진다.

### 단계 2. 목록 레이아웃 전환
- 현재 `customer-summary` 카드 행을 제거하고 테이블/그리드 행으로 전환한다.
- sticky 헤더, hover, selected row, status badge, compact spacing을 적용한다.
- `compareCustomersForList` 우선순위는 기본 정렬로 유지해도 된다.

### 단계 3. 상세 패널 → 드로어 전환
- 현재 항상 우측에 있는 `panel-customer-editor`를 “닫을 수 있는 우측 드로어” 개념으로 바꾼다.
- 빈 상태일 때는 큰 공백 대신 “행 선택 안내”만 간단히 보여준다.
- 새 고객 등록도 같은 드로어 프레임을 사용한다.

### 단계 4. 상세 정보 재배치
- 현재 상세 상단/탭/accordion 구조를 드로어 패턴에 맞게 정리한다.
- “발행 이력”은 드로어 안 탭 또는 섹션 전환으로 유지한다.
- “더보기/삭제/연결 해제”는 맨 아래 위험 구역으로 이동한다.

### 단계 5. 상단 운영 요소 정리
- 고객 탭 상단 칩을 테이블형 화면에 맞게 더 얇고 실무적인 메트릭으로 다듬는다.
- 예: 전체 / 조치 필요 / 발행 가능 / 연결 미완료
- 필요하면 `CustomersTab` 내부 툴바와 `App.tsx` 액션바 책임을 재정리한다.

### 단계 6. 반응형 정리
- 넓은 화면: 목록 + 우측 도킹 드로어
- 중간 화면: 목록 폭 축소 + 드로어 overlay
- 작은 화면: 드로어 full-screen sheet

## 수용 기준
- 고객 목록이 카드 느낌보다 표/관리 콘솔 느낌으로 보인다.
- 한 화면에서 최소 6개 수준의 실무 컬럼을 바로 스캔할 수 있다.
- 행 선택 시 우측 상세가 토글로 열린다.
- 상세가 닫히면 목록이 더 넓게 보인다.
- 기존 핵심 액션(가입/인증서 등록/상태 점검/저장/삭제/이력 확인)이 모두 유지된다.
- 검색/필터/정렬(또는 기본 우선순위)이 깨지지 않는다.
- 새 고객 등록이 별도 화면이 아니라 동일한 드로어 패턴 안에서 동작한다.

- 리스크: 현재 액션바와 고객 탭 툴바가 역할 중복될 수 있음

## 최종 권장안
- **추천 방향은 “테이블 우선 + 우측 도킹/토글 드로어”**다.
- 완전한 모달보다 이 방식이 사용자가 여러 고객을 연속으로 확인하기 쉽고,
  지금 원하는 “데이터 관리 화면” 느낌에도 가장 잘 맞는다.
