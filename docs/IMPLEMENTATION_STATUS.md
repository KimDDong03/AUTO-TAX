# AUTO-TAX 구현 상태

이 문서는 2026-03-27 기준 현재 구현 완료 범위와 남은 작업을 빠르게 확인하기 위한 상태 문서다.

## 1. 현재 구현 완료

- 고객사 작업공간 기반 멀티테넌트 구조
- 운영자 전용 탭과 고객사 개통 흐름
- 운영자 이메일 로그인
- 고객사 `로그인 아이디 + 비밀번호` 로그인
- 공개 회원가입 차단
- 고객사 `owner / member` 권한 분리
- 고객사 내부 사용자 추가/삭제
- 고객사 member 임시 비밀번호 재설정
- owner 임시 비밀번호 운영자 재설정
- 고객사별 메일 설정 저장
- 고객사별 팝빌 운영값 저장
- 팝빌 비밀값 서버 전용 분리
- 관리 고객 등록/수정/삭제
- 발전소명 기준 고객 매칭
- 메일 동기화 수동 실행
- 검수 후 개별 발행
- 검수 대기 전체 발행
- 월 자동 실행일/시각 설정
- 고객별 `검수 후 발행 / 월 자동 발행` 설정
- Supabase 기반 저장 구조
- `job_queue` 기반 배치 구조
- Supabase Edge Function `job-tick`
- Supabase cron 1분 주기 트리거 연결
- 자동 작업 재시도 정책
- 월 자동 처리 요약 메일
- 로그인 화면 `요청 문의` 메일 전송
- Vercel 배포 구조
- `/api/health` 기준 프로덕션 API 정상 응답 확인

## 2. 현재 동작 확인 완료

- Vercel 웹 배포 정상
- Vercel API 응답 정상
- Supabase Edge Function 배포 정상
- Edge Function -> Vercel 내부 큐 API 호출 정상
- Supabase cron job 등록 및 최근 실행 성공
- 원격 마이그레이션 적용 완료

## 3. 아직 남은 작업

- 한전 실전 메일 형식 다양성 검증
- 발전소명 매칭 실패 케이스 보정
- 팝빌 발행 예외 케이스 정리
- 실제 인증서 등록/발행 준비 흐름 검증
- 고객사 설정 화면 문구/구조 마감
- 운영자 화면 세부 운영 기능 보강
- 배치 실패 이력 조회 화면 보강
- 테스트 시나리오 문서화
- 운영 매뉴얼 고도화

## 4. 아직 미구현 또는 후순위

- 공동인증서 갱신 보조 자동화 본 구현
- 별도 Node 워커 분리 운영
- 고객 대량 엑셀 업로드
- 구독/결제 제품 내 UI
- 고객 포털
- 범용 세금계산서 SaaS 확장

## 5. 운영 전 필수 체크리스트

- 고객사 작업공간 생성
- owner 계정 개통
- 고객사 메일 설정 입력
- 고객 등록
- 발전소명 등록
- 팝빌 가입 확인
- 인증서 등록 상태 확인
- 테스트 메일 1건 동기화
- 테스트 초안 1건 수동 발행 확인
- 자동 발행 고객 1곳으로 월 자동 처리 점검
- 실패 알림 메일 수신 확인

## 6. 현재 상태 평가

- 제품 방향: 고정됨
- 핵심 구조: 구현됨
- 배포 구조: 연결됨
- 자동 배치: 동작함
- 운영 준비도: `1차 운영 가능 수준`
- 완성도: `실전 검증과 마감이 더 필요한 상태`

3. 운영자 화면 보강
4. 실제 고객사 온보딩 1건 진행
