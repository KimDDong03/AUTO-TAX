AUTO-TAX 도입 제안서

전자세금계산서 발행 업무의 수작업 감소 및 업무 효율 개선 방안

작성일: 2026.03.26

1. 제안 목적

본 제안서는 현재 전자세금계산서 발행 업무를 보다 쉽고 정확하게 처리하기 위해 AUTO-TAX 프로그램을 도입하는 방안을 설명하기 위해 작성하였습니다.

AUTO-TAX의 목적은 사람이 반복해서 하던 작업을 줄이고, 메일 확인부터 세금계산서 발행 준비까지의 흐름을 한 번에 관리할 수 있도록 만드는 것입니다.

2. 현재 업무 방식

  1. 한전 신재생에너지 요금안내 메일을 확인한다.
  2. 메일에 적힌 공급가액, 부가세, 발전소명 등의 내용을 확인한다.
  3. 해당 내용을 엑셀에 직접 정리한다.
  4. 홈택스에 접속한다.
  5. 고객별 공동인증서로 각각 로그인한다.
  6. 전자세금계산서를 한 건씩 직접 발급한다.

3. 현재 방식의 문제점

  • 메일 확인부터 발급까지 전 과정이 수작업으로 진행됩니다.
  • 엑셀 입력 과정에서 누락이나 오입력 가능성이 있습니다.
  • 고객마다 공동인증서 로그인을 반복해야 해서 시간이 오래 걸립니다.
  • 발행이 몰리는 시기에 업무 부담이 커집니다.
  • 발행 이력이나 예외 상황을 한눈에 보기 어렵습니다.

4. 개선 방향

AUTO-TAX는 기존의 메일 확인, 엑셀 정리, 홈택스 수기 발행 방식에서 벗어나, 메일 내용을 자동으로 정리하고 고객별 세금계산서 초안을 먼저 준비한 뒤, 담당자가 확인 후 발행하는 방식으로 업무를 바꾸는 것을 목표로 합니다.

5. 주요 기능

  • 메일 내용 자동 정리
  • 고객별 세금계산서 초안 자동 작성
  • 고객 정보 및 발전소명 관리
  • 검수 후 개별 발행 또는 일괄 발행
  • 발행 실패 내역 확인
  • 팝빌 가입 상태 확인
  • 인증서 등록 여부 및 만료일 확인
  • 운영자 알림 기능
  • 데이터 백업 기능

6. 1차 도입 범위

  • 메일 확인 자동화
  • 메일 내용 정리
  • 고객 찾기 및 초안 작성
  • 검수 후 발행
  • 고객별 발행 현황 확인
  • 인증서 상태 확인
  • 발행 실패 및 예외 내역 관리

  • 반복 입력 업무 감소
  • 엑셀 정리 시간 감소
  • 오입력 및 누락 위험 감소
  • 고객별 공동인증서 로그인 반복 부담 감소
  • 발행 준비 시간 단축

  • 발행 이력과 예외 상황 관리 용이

8. 진행 방식 제안

  1. 기본 기능 적용: 고객 정보 등록, 메일 확인 자동화, 초안 자동 작성, 검수 후 발행
  2. 운영 안정화: 예외 건 정리, 발행 실패 관리, 인증서 상태 점검
  3. 추가 자동화 검토: 자동 발행, 인증서 갱신 보조, 요약 보고 기능
