# AUTO-TAX IA 초안

> 이 문서는 와이어프레임 작업을 위한 `정보구조 초안`이다.  
> canonical runtime 문서가 아니라, 현재 구현을 바탕으로 화면 구조와 기능 묶음을 제안하는 설계 초안이다.

## 1. 목적

- 현재 AUTO-TAX 기능을 `어떤 화면 단위`로 나눌지 정리한다.
- 사용자 앱과 플랫폼 관리자 앱의 경계를 명확히 한다.
- 와이어프레임 설계 전에 `탭`, `섹션`, `흐름`, `우선순위`를 정리한다.

## 2. IA 기준

### 기준 1. 사용자가 완료하려는 일을 우선한다

- 홈은 `판단과 이동`이 중심이다.
- 발행은 `처리와 확인`이 중심이다.
- 고객은 `비교와 관리`가 중심이다.
- 인증서는 `상태 점검과 연결`이 중심이다.
- 설정은 `운영값 관리`가 중심이다.

### 기준 2. 서로 다른 목적의 앱은 섞지 않는다

- 일반 사용자용 작업공간 앱과 플랫폼 관리자용 운영 앱은 목적이 다르다.
- `ops`는 일반 사용자 흐름의 연장선이 아니라 별도 운영 도메인으로 취급한다.

### 기준 3. 준비 단계와 일상 운영 단계를 구분한다

- 도입 준비는 `초기 운영 가능 상태를 만드는 흐름`이다.
- 홈/발행/고객/인증서는 `운영 이후 반복 사용되는 흐름`이다.

### 기준 4. 인증서는 독립 도메인으로 유지한다

- 인증서는 설정의 하위 보조 기능이 아니라 고객 운영과 발행 준비에 직접 연결되는 핵심 도메인이다.

## 3. 상위 구조

AUTO-TAX는 크게 세 층으로 나눈다.

1. 공개 진입층
2. 작업공간 사용자 앱
3. 플랫폼 관리자 앱

## 4. 공개 진입층

### 4.1 공개 로그인 포털

- 목적: 고객사 사용자의 서비스 진입
- 포함 요소:
  - 로그인 입력
  - 기본 안내
  - 오류/재시도

이 영역은 독립된 단일 목적 화면으로 본다.

## 5. 작업공간 사용자 앱

## 5.1 작업공간 앱의 1차 구조

현재 기준 IA 초안의 1차 메뉴는 다음과 같이 둔다.

- 홈
- 세금계산서 발행
- 고객
- 인증서
- 설정
- 도입 준비

조건부/권한 별도:

- 관리자

## 5.2 1차 구조를 이렇게 나누는 이유

- `홈`은 요약과 우선순위 판단 중심이다.
- `세금계산서 발행`은 실제 운영 처리 집중 영역이다.
- `고객`은 관리 대상 자체를 다루는 영역이다.
- `인증서`는 고객과 발행 사이를 잇는 독립 운영 도메인이다.
- `설정`은 운영값과 계정/사용자 관리를 담당한다.
- `도입 준비`는 초기 세팅 상태를 다시 열어볼 수 있는 별도 허브 역할을 한다.

## 6. 작업공간 앱 세부 IA

## 6.1 홈

### 역할

- 현재 작업공간 상태 요약
- 오늘 우선 처리할 일 제시
- 다른 업무 화면으로의 진입 허브

### 포함할 정보군

- 온보딩/준비 상태
- 우선순위 카드
- 검수 대기 초안
- 최근 수신 메일
- 최근 발행 이력

### 연결 방향

- 발행
- 고객
- 도입 준비
- 설정

### IA 메모

- 홈은 상세 업무 수행 화면이 아니라 `상황판 + 출발점`으로 둔다.
- 홈 안에서 모든 것을 해결하기보다, 다른 도메인으로 자연스럽게 이동시키는 역할을 유지한다.

## 6.2 도입 준비

### 역할

- 초기 세팅과 첫 운영 검증을 단계별로 묶는 허브

### 현재 단계 구조

1. 메일 연결
2. 발행 기본값 입력
3. 로컬 헬퍼 준비
4. 고객 초기 등록
5. 인증서 연결 마무리
6. 첫 메일 동기화
7. 미매칭 메일 예외 처리
8. 첫 발행 확인

### 연결되는 내부 화면/기능

- 설정
- 초기 등록
- 인증서
- 발행

### IA 메모

- 도입 준비는 일반 운영 메뉴와 다르게 `업무 도메인 분류`보다 `순서`가 중요한 허브다.
- 따라서 홈/설정과는 별도 문법을 가질 수 있다.
- 다만 운영 시작 후에도 다시 열어볼 수 있어야 하므로 완전히 숨기는 구조는 피한다.

## 6.3 세금계산서 발행

### 역할

- 발행 대기 초안 처리
- 실패 건 재확인
- 미매칭 메일 후속 처리
- 발행 완료 건 확인

### 내부 정보군

- 상태 필터
  - 검수 대기
  - 자동 대기
  - 발행 중
  - 발행 완료
  - 고객 미매칭
  - 전체
- 목록 영역
  - 초안 목록
  - 미매칭 메일 목록
- 상세 확인 영역
  - 발행 정보
  - 고객 정보
  - 실패 정보
  - 팝빌 확인 정보

### 연결 방향

- 고객
- 홈
- 팝빌 확인 흐름

### IA 메모

- `세금계산서 발행`은 현재 구현 기준에서 독립 1차 메뉴로 유지한다.
- 홈의 검수 대기/예외 메일은 결국 여기로 모이게 된다.
- `발행 처리`와 `발행 후 확인`이 한 도메인에 있지만, 같은 화면군 안에서 단계적으로 다룰 수 있다.

## 6.4 고객

### 역할

- 고객 목록 관리
- 고객별 준비 상태 파악
- 고객 상세 수정
- 고객별 발행 이력 확인

### 내부 정보군

- 검색
- 상태 필터
- 요약 카드
- 고객 목록
- 고객 상세
- 고객 발행 이력
- 위험 작업

### 연결 방향

- 발행
- 인증서
- 초기 등록 결과 확인

### IA 메모

- 고객은 `도메인 객체 중심` 화면이다.
- 발행, 인증서, 팝빌 연동 상태가 이 화면으로 모여들지만, 고객 자체를 기준 축으로 유지한다.

## 6.5 인증서

### 역할

- 로컬 인증서 점검
- 고객과 인증서 연결 관리
- 발급/갱신 준비 상태 확인

### 내부 정보군

- helper 상태
- 읽은 인증서 요약
- 연결된 고객 상태
- 미연결 인증서 목록
- 준비 필요 / 결제 가능 / 만료 임박 상태

### 연결 방향

- 고객
- 설정(helper 상태)
- 도입 준비

### IA 메모

- 인증서는 `설정` 내부 하위 메뉴보다 독립 도메인으로 두는 것이 현재 흐름과 더 맞다.
- 고객과 연결되지만 고객 화면에 흡수되기엔 로컬 장치 상태와 인증서 상태가 별도 축을 이룬다.

## 6.6 설정

### 역할

- 운영값과 계정/사용자 관리

### 내부 2차 섹션

- 메일 연결
- 발행 설정
- 헬퍼 상태
- 계정 / 작업공간

### 포함 기능

- 메일 연결 테스트
- 월간 메일 동기화 일정
- 신규 고객 기본값
- helper 상태 점검
- 본인 비밀번호 변경
- 작업공간 사용자 관리

### 연결 방향

- 도입 준비
- 인증서

### IA 메모

- 설정은 `운영값`, `연결상태`, `계정관리`가 함께 있지만, 모두 “업무를 위한 준비값”이라는 공통점을 가진다.
- 인증서 실무는 설정에서 분리하고, helper 상태 수준까지만 설정에 남긴다.

## 7. 플랫폼 관리자 앱

## 7.1 관리자(ops)

### 역할

- 작업공간 개통
- owner 계정 관리
- 플랫폼 작업 실행
- 파트너 포인트/자동화/로그 모니터링

### 내부 정보군

- 작업공간 생성
- 작업공간 목록
- managed customer limit 관리
- owner 비밀번호 재설정
- 내부 작업 실행
- 파트너 포인트
- renewal agent 상태
- 운영 로그

### IA 메모

- 이 영역은 일반 사용자 앱에서 분리된 백오피스 성격이 강하다.
- 일반 사용자 메뉴와 동일한 정보 구조 문법을 그대로 쓰기보다, 운영 콘솔 문법을 허용한다.

## 8. 앱 간 관계

### 8.1 공개 포털 -> 작업공간 앱

- 공개 로그인 후 작업공간 앱으로 진입한다.

### 8.2 작업공간 앱 내부 핵심 흐름

- 홈 -> 발행
- 홈 -> 고객
- 홈 -> 도입 준비
- 도입 준비 -> 설정 / 초기 등록 / 인증서 / 발행
- 고객 <-> 인증서
- 발행 <-> 고객

### 8.3 작업공간 앱 -> 관리자 앱

- 일반 사용자는 직접 이어지지 않는다.
- platform admin만 관리자 영역에 접근한다.

## 9. IA 초안 기준 화면 지도

### 공개 진입

- 공개 로그인 포털

### 작업공간 앱

- 홈
- 도입 준비
  - 메일 연결 단계
  - 발행 기본값 단계
  - 헬퍼 준비 단계
  - 고객 초기 등록 단계
  - 인증서 연결 마무리 단계
  - 첫 메일 동기화 단계
  - 예외 메일 처리 단계
  - 첫 발행 확인 단계
- 세금계산서 발행
- 고객
- 인증서
- 설정
  - 메일 연결
  - 발행 설정
  - 헬퍼 상태
  - 계정 / 작업공간

### 플랫폼 관리자 앱

- 관리자
  - 작업공간 관리
  - owner 계정 관리
  - 내부 작업
  - 파트너 포인트
  - 자동화/에이전트 상태
  - 로그

## 10. 현재 구현 기준에서 유지할 구조적 판단

- `세금계산서 발행`은 홈의 하위 패널이 아니라 독립 1차 메뉴로 유지한다.
- `인증서`는 설정의 하위 패널이 아니라 독립 1차 메뉴로 유지한다.
- `도입 준비`는 조건부 재진입 가능한 상위 허브로 유지한다.
- `관리자`는 일반 사용자 앱과 분리된 권한 전용 상위 영역으로 유지한다.

## 11. 와이어프레임 단계로 넘길 핵심 질문

- 홈에서 어디까지 요약하고 어디서부터 다른 화면으로 넘길 것인가
- 도입 준비를 얼마나 독립된 경험으로 둘 것인가
- 발행에서 `초안`과 `미매칭 메일`을 어느 정도까지 같은 흐름으로 묶을 것인가
- 고객과 인증서 사이의 상호 이동을 어떤 강도로 드러낼 것인가
- 설정 안에서 상태 요약과 입력 폼의 비중을 어떻게 둘 것인가
- 관리자 콘솔을 일반 앱과 얼마나 다르게 보이게 할 것인가

## 12. 비고

- 이 문서는 현재 구현을 기준으로 한 `초안`이다.
- 이후 와이어프레임 단계에서 병합, 분리, 이름 변경이 일어날 수 있다.
- canonical runtime 문서를 대체하지 않는다.
