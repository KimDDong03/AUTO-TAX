# AUTO-TAX Component Checklist

이 문서는 **화면별 필수 UI 컴포넌트 체크리스트**다.

목적:

- 디자인 단계에서 빠지는 요소를 줄인다
- 구현 단위를 나누기 쉽게 만든다
- `DESIGN.md` 기준으로 일관된 UI를 유지한다

관련 문서:

- `docs/LOW_FI_WIREFRAMES.md`
- `docs/FEATURE_LIST.md`
- `DESIGN.md`

---

## 1. 공통 컴포넌트

모든 메인 화면에서 공통으로 고려할 것:

- `Action Bar`
  - 화면 제목
  - 작업공간/모드 정보
  - 핵심 CTA 1개
  - 상태 chip
- `Panel`
- `chip / status`
- `empty state`
- `loading state`
- `error state`
- `AppDialog`
- 모바일 1열 재배치

공통 체크:

- [ ] primary CTA는 1개인가
- [ ] 빈 상태가 별도 카드처럼 뜨지 않는가
- [ ] 작업 직후 즉시 피드백이 보이는가
- [ ] 위험 액션에 확인 모달이 있는가
- [ ] 상태가 텍스트만이 아니라 chip으로도 보이는가

- [ ] 최근 수신 / 최근 발행 feed panel
- [ ] 메일 동기화 CTA
- [ ] 예외 처리 바로가기

리스트/행 요소:

- [ ] draft row
- [ ] inbox row
- [ ] 발행 결과 row

상태 컴포넌트:

- [ ] 초안 없음 empty
- [ ] 첫 동기화 전 안내
- [ ] 최근 수신 없음 empty
- [ ] 막힘 없음 success 상태

대화상자:

- [ ] 전체 발행 확인 모달

## 12. 최종 체크

디자인 컴프 전 최종 확인:

- [ ] 홈 / 고객 / 인증서 / 설정 / 도입 준비 5화면이 모두 정의되었는가
- [ ] 고객은 `기본 정보 / 발행 이력` 탭 구조인가
- [ ] 인증서는 행 인라인 확장 구조인가
- [ ] 설정은 헬퍼 상태 요약만 포함하는가
- [ ] 인증서 작업은 설정 없이도 독립 화면으로 가능한가
