# AUTO-TAX Feature List

이 문서는 **디자인/기획/개발 공통 기준의 기능 목록**이다.

목적:

- 화면 설계 전에 기능 범위를 고정한다.
- 사이트맵과 와이어프레임의 기준을 맞춘다.
- 구현 우선순위를 `필수 / 확장 / 관리자 전용`으로 나눈다.

기준 문서:

- `docs/PRODUCT_RESHAPE_PLAN.md`
- `docs/WIREFRAME_ERD.md`
- `docs/IMPLEMENTATION.md`

---

## 1. 제품 한 줄 정의

AUTO-TAX는 **한전 정산 메일을 읽고, 고객/인증서 준비 상태를 점검한 뒤, 세금계산서 초안 생성과 발행을 운영하는 작업 도구**다.

핵심 흐름:

1. 작업공간 준비
2. 고객/인증서 준비
3. 메일 동기화
4. 초안 검토/발행
5. 예외 처리

---

## 2. 사용자 유형

## 2-1. 작업공간 사용자

### Owner

- 작업공간 설정 변경
- 멤버 초대/삭제
- 고객/인증서/초안 운영

### Member

- 고객 운영
- 메일 동기화
- 초안 검토/발행
- 인증서 작업

## 2-2. 플랫폼 관리자

- 고객사 작업공간 개통
- 작업공간 한도 조정
- owner 비밀번호 재설정
- 운영 로그/내부 진단 확인

## 2-3. 공개 사용자

- 고객 전용 접속 포털 확인
- 계약 완료 고객 안내 확인
- 로그인

---

## 3. 필수 사용자 기능

아래는 **사용자용 메인 제품 IA** 기준의 필수 기능이다.

## 3-1. 공통 / 앱 셸

- 로그인
- 활성 작업공간 bootstrap
- 좌측 메인 내비게이션
  - 홈
  - 고객
  - 인증서
  - 설정
  - 도입 준비
- 현재 작업공간 표시
- 현재 화면별 핵심 CTA 표시

## 3-2. 홈

목표: **오늘 처리할 일 파악**

필수 기능:

- 오늘 할 일 요약
- 긴급 예외/막힘 우선 노출
- 검토할 초안 목록
- 최근 수신 메일 확인
- 최근 발행 결과 확인
- 메일 동기화 실행 진입
- 고객 화면 / 예외 처리 / 최근 결과로 바로 이동

## 3-3. 고객

목표: **고객 단위로 발행 준비 상태를 해결**

필수 기능:

- 고객 리스트 조회
- 고객 필터
  - 전체
  - 막힘
  - 발행 가능
  - 인증서 점검
  - 연결 필요
- 고객 검색
- 고객 신규 등록
- 고객 정보 수정
- 고객 삭제
- 고객 상세 확인
  - 기본 정보 탭
  - 발행 이력 탭
  - 팝빌 상태
  - 인증서 상태
  - 다음 조치
- 고객별 막힘 사유 체크리스트
- 고객별 해결 액션
  - 팝빌 가입
  - 인증서 등록 URL 열기
  - 인증서 상태 재확인
  - 팝빌 연결 초기화
- 고객별 발행 이력 확인

## 3-4. 인증서

목표: **인증서 작업을 설정과 분리된 운영 작업 공간에서 처리**

필수 기능:

- 로컬 헬퍼 상태 확인
- 로컬 인증서 목록 불러오기
- 인증서 리스트 확인
  - 인증서명
  - 종류
  - 만료일
  - 연결 고객
  - 상태
- 인증서 행 인라인 확장 상세
  - 추천 고객
  - 연결 상태 상세
  - 바로 실행 액션
- 인증서 필터
  - 전체
  - 연결 안 됨
  - 만료 임박
  - 결제 가능
  - 조치 필요
- 고객-인증서 연결
- 고객-인증서 연결 해제
- 갱신 준비 실행
- 결제 페이지 열기
- 추천 연결 대상 확인

## 3-5. 설정

목표: **작업공간 운영 기본값과 계정을 관리**

필수 기능:

### 메일 연결

- 메일 계정 입력
- IMAP/SMTP 연결 테스트
- 알림 수신 메일 설정
- 메일 연결 검증 상태 확인

### 발행 설정

- 기본 발행일/시간
- 메일 수집 주기
- 팝빌 사용자 접두사
- 팝빌 공통 비밀번호
- 담당자 정보
- 갱신 부서/팩스/발급용 비밀번호
- 자동 저장 상태 표시

### 헬퍼 상태

- 헬퍼 연결 여부
- 헬퍼 버전
- 업데이트 필요 여부
- 마지막 확인 시각
- 읽은 인증서 수
- 인증서 읽기 준비 상태 요약
- 상태 새로고침
- 인증서 화면으로 이동
- 인증서 목록/연결/갱신 액션은 포함하지 않음

### 계정 / 작업공간

- 내 비밀번호 변경
- 작업공간 멤버 조회
- 멤버 추가
- 멤버 제거
- 멤버 임시 비밀번호 재설정

## 3-6. 도입 준비

목표: **첫 발행까지의 경로를 제품이 안내**

필수 기능:

- 단계형 onboarding 진입
- 현재 단계 / 완료 단계 / 막힌 이유 표시
- 단계별 실행 액션 제공

필수 단계:

1. 메일 연결
2. 발행 기본값 입력
3. 로컬 헬퍼 준비
4. 고객 초기 등록
5. 인증서 연결 마무리
6. 첫 메일 동기화
7. 미매칭 메일 예외 처리
8. 첫 발행 확인

## 3-7. 고객 초기 등록

목표: **엑셀 기반 고객 등록/반영**

필수 기능:

- 양식 다운로드
- 파일 업로드
- 미리보기
- 생성/갱신/실패 수 확인
- 업로드 경고 확인
- 반영 실행
- 반영 진행률 확인

## 3-8. 예외 메일 처리

목표: **미매칭 메일을 빠르게 고객에 연결**

필수 기능:

- 미매칭 메일 리스트 조회
- 빠른 고객 등록
- 메일 재처리
- 정산월 완료 처리

## 3-9. 초안 / 발행

목표: **초안 검토 후 발행**

필수 기능:

- 초안 목록 조회
- 단건 발행
- 일괄 발행
- 발행 실패 확인
- 발행 취소 후 검수 대기로 되돌리기
- 팝빌 정보 확인
- 보기 URL 열기
- 인쇄 URL 열기

## 4. 관리자 전용 기능 (`ops`)

이 영역은 메인 사용자 IA와 분리된 **플랫폼 관리자 백오피스**다.

필수 기능:

- 개통된 고객사 작업공간 목록
- 새 작업공간 개통
- 관리 고객 한도 변경
- owner 비밀번호 재설정
- 파트너 포인트 확인
- 내부 잡 수동 실행
- 운영 로그 확인
- 로컬 진단/갱신 작업 상태 확인

## 7. 와이어프레임 전에 고정해야 할 것

와이어프레임 착수 전 아래는 고정하는 것이 좋다.

- 메인 메뉴 구조
- 인증서 화면 분리 원칙
- 설정은 헬퍼 상태 요약만 담당하고 인증서 작업은 별도 화면으로 분리
- 도입 준비 8단계
- 홈의 우선순위 블록 구조
- 고객 화면의 리스트 + 상세 + 발행 이력 탭 구조
- 인증서 화면의 인라인 확장행 구조
- 설정에서 인증서 작업을 제외한다는 원칙
