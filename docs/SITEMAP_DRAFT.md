# AUTO-TAX 사이트맵 초안

> 이 문서는 `2026-04-23` 현재 구현과 기능명세/IA 초안을 바탕으로 만든 화면 지도 초안이다.  
> 목적은 화면 범위와 상하위 관계를 한눈에 보는 것이다.

## 1. 문서 목적

- AUTO-TAX 전체 화면 범위를 트리 구조로 본다.
- 공개 진입, 작업공간 사용자 앱, 플랫폼 관리자 앱의 경계를 구분한다.
- 와이어프레임 전 단계에서 `무슨 화면이 있는지`, `어떤 화면이 어떤 상위에 속하는지`를 확인한다.

## 2. 최상위 구조

```text
AUTO-TAX
├─ 공개 진입층
│  └─ 공개 로그인 포털
├─ 작업공간 사용자 앱
│  ├─ 홈
│  ├─ 세금계산서 발행
│  ├─ 고객
│  ├─ 인증서
│  ├─ 설정
│  └─ 도입 준비
└─ 플랫폼 관리자 앱
   └─ 관리자
```

## 3. 공개 진입층

```text
공개 진입층
└─ 공개 로그인 포털
   ├─ 로그인 입력
   ├─ 오류 안내
   └─ 기본 안내
```

## 4. 작업공간 사용자 앱

```text
작업공간 사용자 앱
├─ 홈
│  ├─ 온보딩/준비 상태
│  ├─ 우선순위 카드
│  ├─ 검수 대기 초안
│  ├─ 최근 수신 메일
│  └─ 최근 발행 이력
├─ 세금계산서 발행
│  ├─ 상태 필터
│  │  ├─ 검수 대기
│  │  ├─ 자동 대기
│  │  ├─ 발행 중
│  │  ├─ 발행 완료
│  │  ├─ 고객 미매칭
│  │  └─ 전체
│  ├─ 초안 목록
│  ├─ 미매칭 메일 목록
│  └─ 발행 상세
│     ├─ 발행 정보
│     ├─ 고객 정보
│     ├─ 실패 사유
│     └─ 팝빌 확인 정보
├─ 고객
│  ├─ 검색
│  ├─ 상태 필터
│  ├─ 요약 카드
│  ├─ 고객 목록
│  ├─ 고객 상세
│  │  ├─ 기본 정보
│  │  ├─ 준비 상태
│  │  ├─ 해결 필요 항목
│  │  └─ 위험 작업
│  └─ 고객별 발행 이력
├─ 인증서

│  ├─ 읽은 인증서 요약
│  ├─ 연결된 고객 상태
│  ├─ 미연결 인증서 목록

├─ 설정
│  ├─ 메일 연결
```

## 5. 플랫폼 관리자 앱

```text
플랫폼 관리자 앱
└─ 관리자
   ├─ 작업공간 생성
   ├─ 작업공간 목록
   ├─ managed customer limit 관리
   ├─ owner 비밀번호 재설정
   ├─ 내부 작업 실행
   ├─ 파트너 포인트
   ├─ renewal agent 상태
   └─ 운영 로그
```

## 6. 화면 간 주요 이동 흐름

```text
공개 로그인 포털
  -> 작업공간 앱 진입

작업공간 앱
  홈 -> 세금계산서 발행
  홈 -> 고객
  홈 -> 인증서
  홈 -> 설정
  홈 -> 도입 준비

도입 준비
  -> 설정
  -> 고객 초기 등록
  -> 인증서
  -> 첫 메일 동기화
  -> 세금계산서 발행

세금계산서 발행 <-> 고객
고객 <-> 인증서
설정 <-> 인증서(helper 상태 문맥)

platform admin
  -> 관리자
```

## 7. 현재 구현과 사이트맵의 대응

### 현재 1차 메뉴 기준

```text
홈
세금계산서 발행
고객
인증서
설정
도입 준비
관리자(권한 조건부)
```

### 현재 구현상 주의할 점

- `도입 준비`는 조건부로 다시 열 수 있는 상위 허브다.
- `세금계산서 발행`은 현재 구현에서 독립 탭이다.
- `인증서`는 현재 구현에서 독립 탭이다.
- `관리자`는 platform admin에게만 노출된다.

## 8. 사이트맵 기준 체크 질문

- 공개 로그인 포털은 작업공간 앱과 충분히 분리되어 보이는가
- 도입 준비는 상위 허브로 보아야 하는가, 보조 흐름으로 보아야 하는가
- 인증서는 고객/설정 어느 쪽의 하위가 아니라 독립 도메인으로 읽히는가
- 관리자 영역은 일반 사용자 앱과 다른 앱처럼 읽히는가
