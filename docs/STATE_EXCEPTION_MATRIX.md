# AUTO-TAX State & Exception Matrix

이 문서는 **화면별 상태 / 예외 / 사용자 대응 액션**을 정리한 매트릭스다.

목적:

- 운영 화면에서 빠지기 쉬운 상태를 선제적으로 정의한다
- 와이어프레임과 hi-fi 디자인에서 누락을 막는다
- 구현 시 로딩/에러/빈 상태를 공통 규칙으로 맞춘다

관련 문서:

- `docs/LOW_FI_WIREFRAMES.md`
- `docs/FEATURE_LIST.md`
- `DESIGN.md`

---

## 1. 공통 상태 규칙

상태 분류:

- `default` 정상 기본 상태
- `loading` 읽는 중
- `empty` 데이터 없음
- `success` 문제 없음 / 완료
- `warn` 주의 / 후속 조치 필요
- `danger` 실패 / 막힘 / 중요 예외

공통 원칙:

- 로딩은 spinner보다 **짧은 진행 문구 + 버튼 상태 변화**를 우선
- empty는 별도 카드 대신 같은 표면 안에서 처리
- danger는 왜 막혔는지 + 다음 액션을 함께 제공
- destructive action은 `AppDialog` 확인 모달 사용

| 상황 | 상태 | 사용자에게 보일 것 | 다음 액션 |
|---|---|---|---|
| 저장 대기 | warn | 저장 대기 chip | 대기 |

| 자동 저장 완료 | success | 자동 저장 chip | 계속 작업 |
| 자동 저장 실패 | danger | 저장 실패 chip | 값 수정/재시도 |
| 메일 연결 안 됨 | warn | 연결 테스트 필요 | 메일 연결 테스트 |
| 메일 연결 성공 | success | 준비됨 | 다음 섹션 이동 |
| 헬퍼 정상 | success | 연결됨 / 버전 / 읽은 인증서 수 | 인증서 화면 이동 |
| 헬퍼 재설치 필요 | warn | 재설치 필요 | 설치 가이드 |

- 설정은 인증서 작업대가 아님
- 여기서 제공하는 인증서 관련 액션은 `상태 새로고침`과 `인증서 화면 이동`까지만

## 12. 디자인/구현 결정 고정

- 고객 상세는 `기본 정보 / 발행 이력` 탭 구조
- 인증서 상세는 우측 패널이 아니라 **행 인라인 확장**
- 설정은 헬퍼 상태 **요약만** 담당
- 인증서 실무 작업은 `certificates` 화면으로 분리
