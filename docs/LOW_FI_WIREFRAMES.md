# AUTO-TAX Low-fi Wireframes

이 문서는 **목표 IA 기준의 1차 low-fi 와이어프레임 초안**이다.

목적:

- 실제 디자인 컴프 전에 화면 구조를 고정한다.
- 정보 우선순위, 액션 위치, 리스트/상세 구성을 맞춘다.
- 구현 시 화면 단위를 나누기 쉽게 만든다.

기준 문서:

- `docs/FEATURE_LIST.md`
- `docs/SITEMAP.md`
- `docs/WIREFRAME_ERD.md`
- `DESIGN.md`

주의:

- 이 문서는 **레이아웃/정보구조 초안**이다.
- 시각 스타일, 색, 타이포, 여백 세부값은 디자인 컴프에서 다듬는다.
- 현재 구현과 100% 일치시키기보다 **목표 구조**를 우선한다.

---

## 1. 공통 프레임

모든 메인 화면은 아래 공통 셸을 공유한다.

```text
+-----------------------------------------------------------------------------------+
| Sidebar | Action Bar: 화면명 / 작업공간 / 핵심 CTA / 상태 chip                    |
+-----------------------------------------------------------------------------------+
|                                                                                   |
| Main Content                                                                      |
|                                                                                   |
+-----------------------------------------------------------------------------------+
```

공통 규칙:

- 좌측 사이드바: 홈 / 고객 / 인증서 / 설정 / 도입 준비
- 상단 액션바: 화면 목적 1줄 + CTA 1개 + 상태 chip
- 메인 본문: `Panel` 중심
- 카드 중첩 최소화
- 빈 상태, 오류 상태, 진행 상태를 같은 표면 안에서 처리

## 3. 홈

목표:

- 오늘 처리할 운영 항목을 우선순위대로 보여준다.

핵심 질문:

- 지금 막힌 건 무엇인가?
- 지금 검토할 초안이 있는가?
- 최근 들어온 메일에 예외가 있는가?

## 3-1. 데스크톱 low-fi

```text
+-----------------------------------------------------------------------------------+
| 홈                                   [초안 n건 확인]  [오늘 할 일/막힘/고객/최근결과] |
+-----------------------------------------------------------------------------------+
| 긴급 예외 / 막힘 우선 블록                                                        |
| [예외 메일 n건] [인증서 주의 n건] [중복 의심 n건]                                  |
+--------------------------------------+--------------------------------------------+
| Panel A. 검토 후 발행 큐             | Panel B. 최근 수신 / 최근 발행              |
|--------------------------------------|--------------------------------------------|
| draft row                            | tab: 최근 수신 | 최근 발행                |
| draft row                            | inbox/feed row                            |
| draft row                            | inbox/feed row                            |
| [전체 발행] [상세 보기]              | [예외 처리로 이동] [최근 결과 보기]        |
+--------------------------------------+--------------------------------------------+
```

## 3-2. 구성 요소

- 상단 상태 chip 4개
  - 오늘 할 일
  - 막힘
  - 고객
  - 최근 결과
- 긴급 예외 바
- 검토 후 발행 큐
- 최근 수신 / 최근 발행 전환 feed

## 3-3. 핵심 CTA

- 초안 n건 확인
- 메일 동기화
- 예외 처리 보기

## 3-4. 상태

- 초안 없음
- 첫 메일 동기화 전
- 최근 수신 없음
- 막힘 없음

## 4. 고객

목표:

- 고객 단위로 발행 준비 상태를 점검하고 바로 해결한다.

구조 원칙:

- **좌측 리스트 / 우측 상세**
- 상세에서 **문제 + 해결 액션**을 같이 보여준다.

## 4-1. 데스크톱 low-fi

```text
+-----------------------------------------------------------------------------------+

+-------------------------------+---------------------------------------------------+
| 고객 리스트                    | 고객 상세                                          |
|-------------------------------|---------------------------------------------------|
| 검색                          | 헤더: 고객명 / 사업자번호 / 상태 chip             |
|                               |                                                   |
| customer row                  | 탭: [기본 정보] [발행 이력]                       |
| customer row                  |                                                   |
| customer row                  | Panel 1. 발행 준비 상태                           |
| customer row                  | - 발행 가능 / 발행 불가                           |
| customer row                  | - 막힘 사유 체크리스트                            |
| customer row                  | - [팝빌 가입] [인증서 등록] [상태 재확인]         |
| customer row                  |                                                   |
| customer row                  | Panel 2. 기본 정보                                |
|                               | - 주소 / 업태 / 업종 / 메모                       |
|                               |                                                   |
|                               | Panel 3. 주소 매칭 규칙                           |
|                               | - 매칭 주소 목록                                  |
|                               |                                                   |
|                               | 발행 이력 탭                                      |
|                               | - issued history list                             |
|                               | - related draft list                              |
+-------------------------------+---------------------------------------------------+
```

## 4-3. 핵심 CTA

- 새 고객 등록
- 팝빌 가입
- 인증서 등록 URL 열기
- 인증서 상태 재확인
- 팝빌 연결 초기화

## 4-4. 상태

- 고객 없음
- 막힌 고객 없음
- 발행 가능 고객만 있음
- 상세 미선택

## 5. 인증서

목표:

- 인증서 관련 실무 작업을 설정과 분리된 한 화면에서 처리한다.

핵심 질문:

- 어떤 인증서가 연결되지 않았는가?
- 어떤 인증서가 만료 임박인가?
- 어떤 인증서가 결제/갱신 단계에 들어갈 수 있는가?

## 5-1. 데스크톱 low-fi

```text
+-----------------------------------------------------------------------------------+
| 인증서                                [인증서 불러오기] [헬퍼 새로고침] [상태 chip] |
+-----------------------------------------------------------------------------------+
| 상단 요약                                                                              |
| 헬퍼 연결됨 / 읽은 인증서 n건 / 결제 가능 n건 / 미연결 n건                             |
+-----------------------------------------------------------------------------------+
| 필터 바                                                                               |
| [조치 필요] [전체] [미연결] [만료 임박] [결제 가능] [검색]                            |
+-----------------------------------------------------------------------------------+
| 인증서 리스트                                                                       |
|-----------------------------------------------------------------------------------|
| CN / 종류 / 만료일 / 연결 고객 / 상태                                              |
| row                                                                                |
|   > 확장                                                                          |
|   > 추천 고객 / 연결 고객 / [고객 연결] [연결 해제]                                |
|   > [갱신 준비] [결제 열기]                                                        |
| row                                                                                |
| row                                                                                |
+-----------------------------------------------------------------------------------+
```

## 5-2. 핵심 CTA

- 인증서 불러오기
- 고객 연결
- 연결 해제
- 갱신 준비
- 결제 열기

## 5-3. 상태

- 헬퍼 미연결
- 읽은 인증서 없음
- 조치 필요 없음
- 추천 연결 후보 다수

## 6. 설정

목표:

- 작업공간 기본값과 계정을 관리한다.

원칙:

- 실제 인증서 작업은 여기 넣지 않는다.
- 여기서는 **헬퍼 상태 요약**까지만 보여준다.
- 요약 범위는 `연결 여부 / 버전 / 업데이트 필요 / 마지막 확인 / 읽은 인증서 수`까지만 둔다.

## 6-1. 데스크톱 low-fi

```text
+-----------------------------------------------------------------------------------+
| 설정                                   [다음 설정 열기] [자동 저장 상태 chip]       |
+------------------------------+----------------------------------------------------+
| 준비 상태 사이드바            | 선택 섹션 상세                                    |
|------------------------------|----------------------------------------------------|
| 1. 메일 연결                 | 메일 연결                                          |
| 2. 발행 설정                 | - 메일 주소                                       |
| 3. 헬퍼 상태                 | - 앱 비밀번호                                     |
| 4. 계정/작업공간             | - IMAP/SMTP 연결 테스트                           |
|                              | - 알림 메일                                       |
|                              |                                                   |
|                              | or                                                |
|                              |                                                   |
|                              | 발행 설정                                          |
|                              | - 기본 발행일/시간                                |
|                              | - 팝빌 접두사 / 담당자 / 비밀번호                 |
|                              |                                                   |
|                              | or                                                |
|                              |                                                   |
|                              | 헬퍼 상태                                          |
|                              | - 연결 상태                                        |
|                              | - 버전 / 업데이트 또는 재설치 필요                 |
|                              | - 마지막 확인 시각 / 읽은 인증서 수                |
|                              | - [상태 새로고침] [인증서 화면으로 이동]           |
|                              |                                                   |
|                              | or                                                |
|                              |                                                   |
|                              | 계정 / 작업공간                                    |
|                              | - 내 비밀번호 변경                                |
|                              | - 멤버 목록 / 추가 / 제거 / 비밀번호 재설정       |
+------------------------------+----------------------------------------------------+
```

## 6-2. 핵심 CTA

- 메일 연결 테스트
- 상태 새로고침
- 인증서 화면으로 이동
- 멤버 추가

## 6-3. 상태

- 저장 대기
- 자동 저장 중
- 저장 실패
- 메일 연결 안 됨
- 헬퍼 재설치 필요

## 7. 도입 준비

목표:

- 첫 발행까지의 경로를 단계형으로 안내한다.

구조 원칙:

- 상단 hero
- 단계 strip
- 현재 단계 1개에 집중

## 7-1. 전체 구조 low-fi

```text
+-----------------------------------------------------------------------------------+
| 도입 준비                                 [현재 단계 CTA] [진행률 chip]             |
+-----------------------------------------------------------------------------------+
| Hero                                                                               |
| 진행률 / 지금 할 일 / 막힌 이유                                                    |
+-----------------------------------------------------------------------------------+
| 단계 strip                                                                         |
| [1 메일 연결] [2 발행 기본값] [3 헬퍼] [4 초기 등록] ... [8 첫 발행 확인]          |
+-----------------------------------------------------------------------------------+
| 현재 단계 본문                                                                     |
| 단계 설명 / 필요한 이유 / 현재 상태 / 실행 버튼 / 완료 조건                        |
+-----------------------------------------------------------------------------------+
```

## 8. 모바일 우선 재배치 원칙

- 좌/우 2단 구조는 한 열로 접는다
- 고객:
  - 필터
  - 리스트
  - 선택 상세(기본 정보 / 발행 이력 탭)
  순서로 세로 배치
- 인증서:
  - 상단 상태
  - 필터
  - 리스트
  - 행 인라인 확장
- 설정:
  - 준비 상태 리스트
  - 선택 섹션 상세
- 도입 준비:
  - hero
  - 단계 strip(가로 스크롤)
  - 현재 단계 본문
