# AUTO-TAX Operations Guide

## 1. 최초 실행

1. `.env` 값을 채운다.
2. `npm install`
3. `npm run dev`
4. 브라우저에서 `http://localhost:5173` 접속

## 2. 시스템 설정 확인

설정 화면에서 아래를 확인한다.

- Gmail IMAP 계정
- Gmail 앱 비밀번호
- SMTP 계정
- SMTP 앱 비밀번호
- 운영자 알림 메일
- 운영 담당자명 / 이메일 / 연락처
- 기본 발행일 / 시각
- 팝빌 LinkID / SecretKey
- 팝빌 ID 접두어
- 팝빌 공통 비밀번호
- 테스트 모드 여부

주의:

- 팝빌 공통 비밀번호는 신규 고객용 기준값이다.
- 이미 가입된 기존 고객의 실제 팝빌 비밀번호를 자동 변경하지는 않는다.

권장 기본값:

- 기본 발행일: `25`
- 기본 발행 시: `14`
- 기본 발행 분: `0`
- 메일 폴링 주기: `5`
- 시간대: `Asia/Seoul`

## 3. 일상 운영 순서

### 매월 초 또는 신규 고객 등록 시

1. 고객 등록
2. 발전소명 등록
3. 자동 생성된 팝빌 ID 확인
4. 팝빌 가입
4. 인증서 등록
5. 인증 상태 확인

### 메일 수집 확인

1. `메일 즉시 동기화` 버튼 실행
2. 최근 수신 메일 목록 확인
3. 고객 매칭 실패가 없는지 확인
4. 검수 대기건 생성 여부 확인

### 발행 확인

- 검수 모드 고객은 `지금 발행` 버튼으로 발행
- 자동 발행 고객은 작업공간에서 정한 월 자동 실행일의 메일 동기화 이후 자동으로 발행되고, 실패 시 로그와 알림 메일을 확인
- 사용자가 월 중간에 직접 처리하고 싶으면 `메일 동기화`와 `전체 발행`을 수동으로 실행
- 월 자동 처리에서 실패한 작업은 시스템이 몇 차례 자동 재시도한 뒤 최종 실패로 남긴다.

## 4. 장애 대응

### 메일 수집 실패

가능한 원인:

- Gmail 앱 비밀번호 오류
- Gmail 계정 잠금 또는 보안 경고
- IMAP 계정 입력 오타
- 네트워크 문제

확인 순서:

1. `.env` 또는 설정 화면 값 확인
2. Gmail 계정 로그인 가능 여부 확인
3. 앱 비밀번호 재발급 후 재입력
4. `메일 즉시 동기화` 재시도

### 고객 매칭 실패

원인:

- 메일의 발전소명과 고객 발전소명이 정확히 다름

조치:

1. 메일에서 실제 발전소명 확인
2. 고객 정보의 발전소명 별칭 추가
3. 다시 메일 동기화

### 팝빌 발행 실패

가능한 원인:

- LinkID / SecretKey 오류
- 고객 팝빌 가입 미완료
- 인증서 미등록 또는 만료
- 시스템설정의 팝빌 공통 비밀번호 또는 자동 생성 ID 규칙 불일치

조치:

1. 고객 팝빌 가입 상태 확인
2. 인증 상태 확인
3. 인증서 만료일 확인
4. 시스템설정의 팝빌 ID 접두어 / 공통 비밀번호와 고객 카드의 자동 생성 ID 재검토
5. 검수 화면에서 재발행

## 7. 보안 주의사항

- Gmail 앱 비밀번호는 외부 공유 금지
- 팝빌 SecretKey는 운영자 외 접근 금지
- `.env`는 git에 올리지 않음
- 인증서 파일 원본은 별도 안전 보관

- `server`
- `mail-sync`
- `scheduler`
- `popbill`
- `api`

우선순위 높은 로그:

- `error`
- `warn`
