# AUTO-TAX Customer Onboarding Guide

## 목적

새 고객을 AUTO-TAX에 등록하고, 세금계산서 자동화가 가능한 상태까지 만드는 절차를 정리한다.

## 1. 사전 준비물

고객별로 아래 정보가 필요하다.

- 사업자번호
- 상호
- 대표자
- 주소
- 업태
- 업종
- 발전소명
- 고객 인증서 등록을 진행할 수 있는 환경

시스템에서 미리 정해둘 값:

- 팝빌 사용자 ID 접두어 예: `HAE_`
- 신규 고객용 팝빌 공통 비밀번호
- 운영 담당자명 / 이메일 / 연락처

## 2. 고객 등록

고객관리 화면에서 아래를 입력한다.

- 고객명
- 사업자번호
- 상호
- 대표자
- 주소
- 업태
- 업종
- 발행 모드
- 발행 일정
- 발전소명

중요:

- 발전소명은 메일 본문 값과 최대한 동일하게 입력한다.
- 여러 표현이 가능하면 발전소명을 여러 줄로 등록한다.
- 고객 저장 시 팝빌 ID는 `접두어 + 고객번호` 형식으로 자동 생성된다.
- 팝빌 비밀번호는 시스템설정의 공통 비밀번호가 자동으로 저장된다.
  이미 만들어진 계정의 실제 비밀번호를 자동 변경하지는 않는다.
- 팝빌 가입 시 담당자 정보는 고객별이 아니라 시스템설정의 운영 담당자 정보를 사용한다.

## 3. 팝빌 연동회원 가입

고객 카드에서 `팝빌 가입` 버튼을 누른다.

성공 기준:

- 고객의 `popbillState`가 `joined`로 바뀐다.

## 4. 인증서 등록

1. 고객 카드에서 `인증서 등록` 버튼 클릭
2. 팝업 URL 오픈
3. 운영자가 고객 인증서를 1회 등록
4. `인증 상태 확인` 버튼 클릭

성공 기준:

- `인증완료` 표시
- 인증서 만료일 조회 가능

## 5. 첫 운영 모드

초기 권장값:

- 발행 모드: `검수 후 발행`
- 최소 2개월 안정화 후 자동발행 전환

## 6. 테스트

고객 등록 후 아래를 확인한다.

1. 메일 즉시 동기화
2. 발전소명 매칭 성공 여부
3. 초안 생성 여부
4. 검수 화면에서 발행 가능 여부

## 7. 자동발행 전환 기준

권장 기준:

- 2개월 이상 수동 검수에서 문제 없음
- 발전소명 매칭 오류 없음
- 인증서 정상
- 팝빌 발행 정상

그 뒤 고객의 발행 모드를 `자동 발행`으로 변경한다.

## 8. 자주 발생하는 문제

### 발전소명 불일치

- 메일: `이상택태양광`
- 고객 등록: `이상택 발전소`

이 경우 매칭 실패가 나므로 별칭을 추가해야 한다.

### 팝빌 인증서 만료

- 인증 상태 확인에서 실패하거나 만료일이 지났으면 인증서를 다시 등록한다.

### 고객 팝빌 정보 확인

- 팝빌 가입 실패 시 시스템설정의 `팝빌 ID 접두어`, `팝빌 공통 비밀번호`, 고객 카드의 자동 생성 ID를 확인한다.
