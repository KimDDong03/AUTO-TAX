# AUTO-TAX 도입 제안서

## 1. 제안 목적

본 제안서는 현재 전자세금계산서 발행 업무를 보다 쉽고 정확하게 처리하기 위해 `AUTO-TAX` 프로그램을 도입하는 방안을 설명하기 위해 작성하였습니다.

AUTO-TAX의 목적은 사람이 반복해서 하던 작업을 줄이고, 메일 확인부터 세금계산서 발행 준비까지의 흐름을 한 번에 관리할 수 있도록 만드는 것입니다.

## 2. 현재 업무 방식

현재는 아래와 같은 방식으로 업무를 처리하고 있습니다.

1. 한전 `신재생에너지 요금안내` 메일을 확인한다.
2. 메일에 적힌 공급가액, 부가세, 발전소명 등의 내용을 확인한다.
3. 해당 내용을 엑셀에 직접 정리한다.
4. 홈택스에 접속한다.
5. 고객별 공동인증서로 각각 로그인한다.
6. 전자세금계산서를 한 건씩 직접 발급한다.

이 방식은 처음에는 가능하지만, 고객 수가 늘어날수록 아래와 같은 불편이 커집니다.

- 메일 확인부터 발급까지 전 과정이 수작업으로 진행됨
- 엑셀 입력 과정에서 누락이나 오입력 가능성이 있음
- 고객마다 공동인증서 로그인을 반복해야 해서 시간이 오래 걸림
- 담당자에 따라 처리 속도와 정확도가 달라질 수 있음
- 발행이 몰리는 시기에 업무 부담이 커짐
- 발행 이력이나 예외 상황을 한눈에 보기 어려움

## 3. 개선 방향

AUTO-TAX는 기존의 `메일 확인 -> 엑셀 정리 -> 홈택스 수기 발행` 방식에서 벗어나, 아래 흐름으로 바꾸는 것을 목표로 합니다.

1. 메일을 자동으로 읽어온다.
2. 필요한 내용을 자동으로 정리한다.
3. 고객별 세금계산서 초안을 자동으로 만든다.
4. 담당자가 화면에서 내용을 확인한다.
5. 확인이 끝난 건만 발행한다.

즉, 사람이 처음부터 끝까지 직접 입력하는 방식이 아니라, 프로그램이 먼저 정리해주고 담당자는 확인과 최종 처리에 집중하는 방식입니다.

## 4. 제안 내용

AUTO-TAX는 전자세금계산서 발행 업무를 보다 편하게 하기 위한 업무용 프로그램입니다.

주요 기능은 다음과 같습니다.

- 한전 요금안내 메일 자동 확인
- 메일 내용 자동 정리
- 고객별 세금계산서 초안 자동 작성
- 고객 정보 및 발전소명 관리
- 검수 후 개별 발행 또는 일괄 발행
- 발행 실패 내역 확인
- 팝빌 가입 상태 확인
- 인증서 등록 여부 및 만료일 확인
- 운영자 알림 기능
- 데이터 백업 기능

## 5. 1차 도입 범위

우선 1차로는 실제 업무에 바로 도움이 되는 범위부터 도입하는 것이 적절합니다.

1차 도입 범위는 아래와 같습니다.

- 메일 확인 자동화
- 메일 내용 정리
- 고객 찾기 및 초안 작성
- 검수 후 발행
- 고객별 발행 현황 확인
- 인증서 상태 확인
- 발행 실패 및 예외 내역 관리

핵심은 처음부터 전부 자동으로 발행하는 것이 아니라, `자동으로 초안을 준비하고 사람은 확인 후 발행하는 방식`으로 안정적으로 시작하는 것입니다.

## 6. 추가 제안 사항

운영이 안정되면 아래 기능도 단계적으로 검토할 수 있습니다.

- 고객 정보 엑셀 일괄 등록
- 자주 틀리는 발전소명 보정 기능
- 자동 발행 기능
- 발행 완료/실패 요약 보고
- 인증서 만료 예정 고객 알림 강화
- 인증서 갱신 보조 기능

이렇게 단계적으로 넓혀가면 처음부터 부담이 크지 않으면서도 실제 업무 효율은 꾸준히 높일 수 있습니다.

## 7. 기대 효과

AUTO-TAX를 도입하면 다음과 같은 효과를 기대할 수 있습니다.

- 반복 입력 업무 감소
- 엑셀 정리 시간 감소
- 오입력 및 누락 위험 감소
- 고객별 공동인증서 로그인 반복 부담 감소
- 발행 준비 시간 단축
- 담당자 업무 부담 완화
- 발행 이력과 예외 상황 관리 용이
- 인증서 상태를 미리 확인하여 발행 차질 예방

## 8. 진행 방식 제안

도입은 아래 순서로 진행하는 것이 현실적입니다.

### 1단계. 기본 기능 적용

- 고객 정보 등록
- 메일 확인 자동화
- 초안 자동 작성
- 검수 후 발행

### 2단계. 운영 안정화

- 예외 건 정리
- 발행 실패 관리
- 인증서 상태 점검

### 3단계. 추가 자동화 검토

- 자동 발행
- 인증서 갱신 보조
- 요약 보고 기능

## 9. 회사 협조 사항

원활한 진행을 위해 아래 사항이 필요합니다.

- 메일 수신 계정 정보
- 팝빌 사용 정보
- 운영 담당자 정보
- 고객 기본 정보
- 발전소명 목록 정리
- 인증서 등록 및 확인을 위한 운영 환경

## 10. 결론

AUTO-TAX는 단순히 세금계산서를 발행하는 프로그램이 아니라, 기존에 사람이 반복하던 전자세금계산서 업무를 더 쉽고 정확하게 처리할 수 있도록 돕는 업무 개선 도구입니다.

특히 현재처럼 메일 내용을 보고 엑셀에 정리한 뒤 홈택스에서 고객별 공동인증서로 로그인하여 하나씩 발행하던 방식에 비해, 업무 시간은 줄이고 실수 위험은 낮출 수 있습니다.

따라서 본 제안은 단순한 프로그램 도입이 아니라, 전자세금계산서 발행 업무를 보다 체계적이고 효율적으로 바꾸기 위한 개선안으로 검토할 가치가 있습니다.
